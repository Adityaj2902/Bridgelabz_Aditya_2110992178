let vs var vs const in JavaScript

1. Declaration and Assignment

var is the oldest keyword to declare variables. It allows both re-declaration and re-assignment within the same scope.

Example with var:

var x = 10;
var x = 20;  // Re-declaration is allowed
console.log(x);  // Output: 20

let was introduced in ES6 and allows re-assignment, but does not allow re-declaration in the same scope.

Example with let:

let y = 30;
y = 40;  // Re-assignment is allowed
console.log(y); // Output: 40

let y = 50;  // SyntaxError: Identifier 'y' has already been declared

const also introduced in ES6 is used for values that should not be re-assigned. It must be initialized at the time of declaration.

Example with const:

const z = 60;
z = 70;  // TypeError: Assignment to constant variable.

2. Scoping

var is function-scoped (or globally scoped if declared outside any function). This can sometimes cause unintentional issues with variable leakage.

Example with var:

function example() {
    if (true) {
        var a = 100;  // 'a' is function-scoped
    }
    console.log(a);  // Output: 100
}
example();

let is block-scoped, which means it is confined to the nearest set of curly braces (`{}`), including loops and conditionals.

Example with let:

function example() {
    if (true) {
        let b = 200;  // 'b' is block-scoped
        console.log(b);  // Output: 200
    }
    console.log(b);  // ReferenceError: b is not defined
}
example();

const behaves the same as let in terms of scoping, i.e., it is also block-scoped.

Example with const:

function example() {
    if (true) {
        const c = 300;  // 'c' is block-scoped
        console.log(c);  // Output: 300
    }
    console.log(c);  // ReferenceError: c is not defined
}
example();

3. Hoisting

var declarations are hoisted to the top of their function or global scope. However, the value is set to undefined until the actual assignment.

Example with var:

console.log(x);  // Output: undefined (due to hoisting)
var x = 10;

let and const are hoisted to the top of their block, but are placed in the temporal dead zone (TDZ) until the declaration is encountered in code. Trying to access them before declaration results in a ReferenceError.

Example with let:

console.log(y);  // ReferenceError: Cannot access 'y' before initialization
let y = 20;

Example with const:

console.log(z);  // ReferenceError: Cannot access 'z' before initialization
const z = 30;

4. Re-assignment

var allows re-assignment at any time.

Example with var:

var a = 10;
a = 20;  // Re-assignment is allowed
console.log(a);  // Output: 20

let also allows re-assignment.

Example with let:

let b = 30;
b = 40;  // Re-assignment is allowed
console.log(b);  // Output: 40

const does not allow re-assignment after initialization.

Example with const:

const c = 50;
c = 60;  // TypeError: Assignment to constant variable

5. Best Practices & Use Cases

var is **outdated** and should generally be avoided in modern JavaScript code due to its unpredictable scoping behavior. It is mostly used for maintaining older codebases.

When to use var:

// Legacy code or function-wide scope
var x = 5;
function test() {
var x = 10;  // Scoped to the function
console.log(x); // Output: 10
}
test();
console.log(x); // Output: 5

let is a safer option when you need a variable that can change within a specific block scope, like loops or conditional blocks.

When to use let:

// When you need a block-scoped variable that can be reassigned
let count = 0;
if (true) {
let count = 1;  // Block-scoped
console.log(count); // Output: 1
}
console.log(count); // Output: 0

const should be used as the default choice for declaring variables whose values should not change after initialization. It helps prevent accidental reassignments, improving code reliability.

When to use const:

// For values that should remain constant
const MAX_ITEMS = 100;
const user = { name: "John", age: 30 };

// You can modify properties of objects declared with const
user.age = 31;
console.log(user.age);  // Output: 31

// But you cannot reassign the variable
user = {};  // TypeError: Assignment to constant variable

6. Comparison Summary

Feature var let const
Scope Function-scoped or globally scoped Block-scoped Block-scoped
Re-declaration Allowed in the same scope Not allowed in the same scope Not allowed in the same scope
Re-assignment Allowed Allowed Not allowed
Hoisting Hoisted with undefined Hoisted to the top but in Temporal Dead Zone (TDZ) Hoisted to the top but in Temporal Dead Zone (TDZ)
Best Use Case Legacy code, or function-wide variables Variables that change within a block scope Variables that should never be reassigned

Note: For most modern JavaScript development, prefer const for declaring variables that shouldn't change, and use let only when re-assignment is required. Avoid var due to its unpredictable scoping behavior.
